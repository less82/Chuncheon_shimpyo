# 쉼표 정류장 운영 환경 분리 기준

## 운영 주소

| 영역 | 권장 주소 | 공개 여부 | 배포 산출물 |
|---|---|---|---|
| 시민 앱 | `https://app.shimpyo.chuncheon.go.kr` | 공개 | `app/dist/citizen` |
| 관리자 웹 | `https://admin.shimpyo.chuncheon.go.kr` | 비공개 | `app/dist/admin` |
| 공개 API | `https://api.shimpyo.chuncheon.go.kr` | 조회·민원 접수만 | 별도 서버 |
| 관리 API | 관리자 도메인의 `/api` 프록시 뒤 | 관리자 인증 필수 | 별도 서버·외부 직접 노출 금지 |
| 인증 | 관리자 도메인의 `/auth` 프록시 뒤 | 관리자 인증 | 별도 서버 |

관리자 웹은 시민 앱의 하위 경로로 운영하지 않는다. `/admin`은 로컬 개발 편의를 위한 주소일 뿐이다.

## 빌드와 배포

```bash
npm run build:citizen
npm run build:admin
```

두 산출물을 같은 웹 서버 루트에 복사하지 않는다. 서로 다른 배포 작업, 서비스 계정, 저장소 버킷과 CDN 배포를 사용한다. 시민 앱의 PWA 서비스 워커는 관리자 도메인에 설치하지 않는다.

## 환경변수

- 시민 배포는 `.env.citizen.example`을 기준으로 배포 시스템에 값을 주입한다.
- 관리자 배포는 `.env.admin.example`을 기준으로 별도 주입한다.
- 관리자 브라우저는 동일 출처의 `/api`, `/auth`만 호출한다. 웹 게이트웨이가 내부 관리 API와 인증 서버로 역방향 프록시한다.
- `VITE_` 변수는 브라우저 번들에 공개된다. API 비밀키, DB 비밀번호, 세션 서명키, OAuth client secret을 절대 넣지 않는다.
- 비밀값은 API·인증 서버의 비밀 저장소에서만 관리한다.

## 인증과 권한의 필수 조건

1. 관리자 웹 진입 전에 기관 계정 인증과 MFA를 요구한다.
2. 관리 API는 모든 요청에서 서버 세션과 역할을 검증한다.
3. 권한은 최소 `조회자`, `처리자`, `승인자`, `시스템관리자`로 분리한다.
4. 민원 열람·상태 변경·내보내기에 사용자, 시간, 대상, 변경 전후 값을 감사 로그로 남긴다.
5. 관리자 세션 쿠키는 `Secure`, `HttpOnly`, `SameSite=Strict`로 발급한다.
6. 상태 변경 요청에는 CSRF 방어를 적용한다.
7. 관리자 도메인은 기관망·VPN·허용 IP 적용 여부를 보안 담당자와 결정한다.

## 현재 구현에서 운영 전 교체할 항목

- 민원 저장과 상태 변경이 현재 브라우저 `localStorage`를 사용한다. 이는 데모 전용이며 운영 데이터 저장소가 아니다.
- 시민 민원 접수는 공개 API와 DB 저장으로 교체해야 한다.
- 관리자 목록 조회와 상태 변경은 인증된 관리 API로 교체해야 한다.
- 관리자 인증 서버와 역할 권한 검사는 아직 구현되지 않았다.
- TAGO 키를 브라우저에 넣지 말고 공개 API 서버가 대신 호출하도록 변경해야 한다.

위 항목이 완료되지 않은 빌드는 실제 행정 운영에 배포하면 안 된다.
