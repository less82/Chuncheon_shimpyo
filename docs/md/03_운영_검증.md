# 03. 운영·보안·검증 기준

> 배포 구조, 보안, 개인정보, 테스트와 남은 업무의 단일 기준이다.

## 운영 구조

- **시민 앱**
  - 권장 주소: `app.shimpyo.chuncheon.go.kr`
  - 배포: 공개 정적 앱
- **시민 API**
  - 권장 주소: `api.shimpyo.chuncheon.go.kr`
  - 배포: 공개 API 서버
- **관리자 웹**
  - 권장 주소: `admin.shimpyo.chuncheon.go.kr`
  - 배포: 별도 비공개 배포
- **관리자 API·인증**
  - 권장 주소: 관리자 게이트웨이의 `/api`, `/auth`
  - 배포: 인증된 내부 서버

시민 앱과 관리자 웹은 HTML, 도메인, 배포 계정, 저장소, 서비스 워커를 분리한다. 관리 API는 시민 API와 게이트웨이·서버 계정·DB 계정을 분리한다.

## 비밀과 인증

- `VITE_` 변수는 공개된다. 비밀키를 넣지 않는다.
- TAGO·지도·경로 API 키는 서버 비밀 저장소에 둔다.
- 관리자 인증은 기관 계정과 MFA를 사용한다.
- 역할은 조회자·처리자·승인자·시스템관리자로 나눈다.
- 세션 쿠키는 `Secure`, `HttpOnly`, `SameSite=Strict`를 사용한다.
- 상태 변경에 CSRF 방어를 적용한다.
- 열람·변경·내보내기는 변경 전후 값을 감사 로그로 남긴다.
- 업로드는 확장자·MIME·용량 검사와 악성파일 검사를 거친다.

## 개인정보

- 정확한 GPS는 경로 계산 후 장기 저장하지 않는다.
- 목적지 상세 검색어는 기본적으로 보관하지 않는다.
- 음성은 텍스트 변환 후 폐기한다. 원본 저장은 별도 동의가 필요하다.
- 자유서술의 전화번호·주소·얼굴·차량번호를 마스킹한다.
- 영구 기기 ID와 지속 위치추적을 사용하지 않는다.
- 공개 집계는 소수 조합을 억제한다.

## 현재 운영 불가 항목

- TAGO 키를 브라우저에서 사용
- 장소·도보 공개 데모 서버 직접 호출
- 자체 로컬 경로 엔진 사용
- 즐겨찾기·제보·처리 상태를 `localStorage`에 저장
- 관리자 인증·역할·감사 로그 서버 없음

이 상태는 시연용이며 실제 행정 시스템으로 배포하면 안 된다.

## 검증

### 시민 사용성

65세 이상 5명 이상이 도움 없이 다음 과업을 수행한다.

1. 앱에서 버스 선택
2. 출발지·목적지 입력
3. 지도에서 장소 확인
4. 탑승 정류장·방면·도착시간 확인
5. 즐겨찾기 또는 정류장 상태 알리기

과업 성공률, 완료시간, 도움 요청, 오선택, 음성 재시도를 기록한다.

### 경로·데이터

- 맞은편 정류장 5쌍 이상 검증
- 대표 출발–목적지 10건 현장 대조
- 버스가 도보시간보다 먼저 도착하는 후보 제외
- 동명 정류장 자동 확정 0건
- 미확인을 없음으로 바꾼 사례 0건
- API 실패를 추정값으로 감춘 사례 0건

### AI·행정

- 시설·문제유형 정밀도와 재현율 측정
- 개인정보 마스킹 확인
- 잘못된 정류장 자동 확정 0건
- 사람 승인 전 시설 상태 변경 0건
- 원문·근거·불확실성·모델 버전 표시
- 제보 접수부터 시민 화면 갱신까지 감사 로그 확인

### 화면·보안

- 320×461px 실제 사용영역에서 핵심 흐름 무스크롤
- iOS·Android 안전영역 충돌 없음
- 200% 확대, 키보드, 스크린리더, 긴 문구 확인
- 타입검사·린트·단위 테스트·빌드 통과
- 비밀키 번들·저장소 포함 0건
- 의존성·CSP·XSS·CSRF·업로드 보안 점검
