# 04. 발표 기준

> 심사 설명과 데모 순서의 단일 기준이다.

## 한 문장

**목적지행 버스 도착정보를 쉽게 보여주고, 기다리는 시민의 말을 검증 가능한 정류장 업무로 바꿉니다.**

## 차별점

```text
시민 편익 → 정류장 상태 신호 → AI 구조화·공식자료 대조
→ 담당자 확인 → 데이터 갱신 → 시민 화면 반영
```

강점은 QR, 음성, 지도 각각이 아니라 이 순환을 끝까지 연결하는 데 있다.

## 데모 순서

1. 설치 앱에서 즐겨찾기 또는 `버스`를 누른다.
2. 출발 위치와 목적지를 말한다.
3. 지도에서 두 장소를 확인한다.
4. 목적지행 버스의 탑승 정류장·방면·도착시간을 본다.
5. 해당 정류장의 상태를 알린다.
6. AI 구조화 결과를 시민이 확인하고 제출한다.
7. 관리자가 공식자료와 중복 신호를 확인한다.
8. 담당자 확정 결과가 시민 화면에 반영된다.

## 반드시 보여줄 근거

- 정류장 관리번호–외부 경로 ID–TAGO 노드 ID 연결
- 실시간 도착정보의 출처와 생성시각
- AI 결과와 사람 확정의 구분
- 고령 사용자 과업 결과
- 현재 구현과 미구현 기능의 구분

## 예상 질문

| 질문 | 답변 |
|---|---|
| 기존 교통 앱과 다른가? | 도착정보와 정류장 상태의 행정 환류를 연결한다. |
| AI가 필요한가? | 자유발화를 구조화하고 중복·자료 충돌을 설명한다. |
| AI가 틀리면? | 시민 확인과 담당자 승인 전에는 상태를 바꾸지 않는다. |
| 조회량이 승차량인가? | 아니다. 서비스 조회 신호일 뿐이다. |
| 고령자가 실제 사용했나? | 별도 사용성 시험 결과로만 답한다. |
| 제보가 민원인가? | 공식 연계 전에는 검수 전 시민 신호다. |
| 맞은편 정류장을 틀리지 않나? | ID·좌표·방면·정차순서를 함께 검증한다. |

## 금지 표현

- 미구현 기능을 완료로 설명
- 제보량을 수혜자·고령자 수요로 표현
- AI가 공사·예산을 결정한다고 표현
- 미확인을 없음으로 표현
- 외부 지도 참고값을 공식 원본으로 표현
- 실시간 실패를 배차간격으로 대체

## 마지막 문장

**쉼표 정류장은 버스 정보를 보는 순간과 행정이 정류장을 고치는 과정을 하나의 데이터 흐름으로 연결합니다.**
