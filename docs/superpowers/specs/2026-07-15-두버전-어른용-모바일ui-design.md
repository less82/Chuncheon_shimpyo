# 쉼표 정류장 — 두 버전(어른용/일반) + 모바일 UI 설계

작성일: 2026-07-15

## 배경 / 목표

현재 시민앱은 코드상 이미 고령자 지향(기본 19px, 터치 48px, 바텀시트)이지만, 데스크톱 넓은 화면에서 요소가 퍼져 "웹 구조"처럼 보이고, 지도 탭으로 정류장을 고르는 방식은 고령자에게 어렵다.

두 가지를 해결한다.

1. **모바일 앱 느낌** — 폰 폭 셸로 감싸 데스크톱에서도 폰처럼 보이고, 터치 위주로 동작.
2. **두 버전(어른용/일반)** — 은행앱 "큰글씨" 모드처럼, 레이아웃까지 다른 두 모드를 언제든 전환.

## 확정 요구사항 (브레인스토밍 결과)

- **버전 모델**: 단순 폰트 배율이 아니라 **레이아웃까지 다른 두 모드**.
  - 일반 = 지도 중심(밀도 높음), 어른용 = 큰 카드/리스트 중심(극단 단순화).
- **어른용 정류장 선택**: **즐겨찾기("내 정류장") 중심**. 등록이 없으면 GPS 최근접으로 폴백.
- **기본/전환**: **어른용이 기본**. 상단에 항상 버전 토글(어른용→"일반", 일반→"큰글씨"). 선택은 localStorage 기억.
- **적용 범위**: 시민 화면 전체(첫 화면, 버스 타러, 즐겨찾기). **행정 대시보드(`/admin`)·인쇄 포스터(`/print/:id`)는 손대지 않음.**

## 불변 규칙 (두 버전 공통 — 프로젝트 절대 규칙)

- 시설은 **3상태(있음/없음/미확인)** 로만 표시하고 항상 아이콘+한글 병기. 데이터 없음을 "없음"으로 단정하지 않는다.
- 데이터 **기준일** 표시 유지.
- 날씨/도착정보 API가 죽어도 지도·시설 표시는 동작(기능 독립성).
- 두 모드는 데이터 로직(`useStops`·`useFavorites`·`arrivals`·`walking`)과 3상태 계약(`facilityText`·`FacilityBadge`)을 **공유**한다. 배치만 다르다.

## 아키텍처 (선택: A)

**A. 뷰모드 스토어 + 모드별 레이아웃 컴포넌트.**
`useViewMode` zustand 스토어 하나(`'elder' | 'normal'`, 기본 `elder`, localStorage 영속)를 최상단에 두고, 시민 라우트에서 모드에 따라 다른 레이아웃을 렌더한다. 데이터/원자 컴포넌트는 공유하고 배치만 다르게 하여, 3상태·기준일·데이터 계약이 한 곳에 유지된다.

기각한 대안:
- B. 같은 DOM + CSS 밀도 전환 — "지도 최소화·즐겨찾기 우선" 같은 구조 변경엔 부적합.
- C. 모드별 라우트 분리 — 라우팅/상태 공유가 번거로워 과함.

## 컴포넌트 설계

### 1. `store/useViewMode.ts` (신규)
- 기존 `useFavorites` 패턴을 그대로 따른다(zustand + localStorage read/write, try/catch 무시).
- localStorage 키: `swimpyo:viewMode`.
- 상태: `mode: 'elder' | 'normal'` (초기값: 저장값 없으면 `'elder'`).
- 액션: `setMode(m)`, `toggle()` (elder↔normal). 변경 시 localStorage 기록.
- 의존: zustand. 다른 모듈에 의존하지 않는 순수 상태 단위.

### 2. `components/VersionToggle.tsx` (신규)
- `useViewMode`를 읽어 큰 pill 버튼 하나를 렌더. 터치타깃 ≥48px.
- 라벨: 현재 `elder`면 "일반"(누르면 normal로), `normal`이면 "큰글씨"(누르면 elder로).
- `aria-label`로 "글자 크게 보기 / 일반 화면으로" 명확히.
- 입력: 없음(스토어 직접 구독). 출력: 클릭 시 `toggle()`.

### 3. 모바일 셸 (App 루트 + CSS)
- 앱 전체를 `.app-shell`(max-width ~480px, 가운데 정렬, min-height 100dvh)로 감싼다.
- 넓은 화면에서 좌우 여백 + 은은한 그림자로 "폰" 느낌. 지도·시트는 셸 폭 안에서 full-bleed.
- `index.css`/신규 셸 CSS에 정의. 기존 `CitizenHome.css`의 `@media (min-width:640px)` 사이드패널 분기는 셸 폭 안에서 자연히 무력화되므로 제거/조정.

### 4. `features/citizen/ElderHome.tsx` (+ `ElderHome.css`) (신규) — 핵심
어른용 첫 화면. 프롭 없음(라우트 엘리먼트).
- **주 정류장 결정 로직**:
  1. `useFavorites().ids` 중 `useStops`로 해석되는 첫 정류장을 "내 정류장"으로.
  2. 즐겨찾기가 없거나 해석 실패 시 → GPS 최근접(거부 시 `CITY_CENTER`) 정류장으로 폴백.
  3. 사용자가 근처 목록에서 다른 정류장을 탭하면 로컬 `selectedId`로 override.
- **구성**:
  - 상단 바: 브랜드 + `VersionToggle`.
  - **내 정류장 큰 카드**: 정류장명(`--fs-2xl`), 4시설 3상태 배지 크게(그늘/의자/조명/도착안내기), 도착정보, 도보시간, `FavoriteStar`, **"버스 타러 →" 큰 버튼**(`Link to="/go"`). 기준일 표기 포함.
  - **"근처 정류장" 큰 목록**: GPS 기준 가까운 순 N개(기본 5). 각 행: 이름 + 거리 + 한 줄 시설 요약(`facilitySummary`). 탭 시 주 카드로 승격.
  - **"지도 보기"** 접이 버튼: 기본 접힘, 펼치면 `MapView` 표시(선택 시 카드 갱신).
- **재사용**: `FacilityBadge`, `arrivals`(`headwayFallback`/`getArrival`), `walking`(`straightWalk`/`getWalkRoute`), `geo`(거리계산), `useStops`, `useFavorites`, `CITY_CENTER`.
- **폴백/독립성**: 도착·도보는 폴백 문구 즉시 표시(스피너 금지), 실시간 오면 갱신. GPS 거부해도 동작.

### 5. `features/citizen/CitizenHome.tsx` (일반 모드) (수정)
- 동작은 유지(지도 중심 + 바텀시트). 상단 바에 `VersionToggle` 추가.
- 데스크톱에서 3버튼이 퍼지던 웹 느낌은 모바일 셸 폭 고정으로 해소.

### 6. `App.tsx` (수정)
- `/` 라우트: `mode === 'elder' ? <ElderHome/> : <CitizenHome/>`.
- `/go`(TripView), `/favorites`(Favorites): 각 컴포넌트가 `useViewMode`를 읽어 어른용/일반 레이아웃 분기(아래).
- `/print/:id`, `/admin`: **변경 없음**.
- 모바일 셸 래퍼를 여기(또는 `main.tsx`)에 적용.

### 7. `features/trip/TripView.tsx` (수정)
- `useViewMode`를 읽어 어른용일 때 큰 글씨·단일 컬럼 단계형 레이아웃(`data-mode="elder"` + CSS). 일반은 기존.
- 데이터/길찾기 로직은 불변.

### 8. `features/citizen/Favorites.tsx` (수정)
- 어른용 홈이 이미 즐겨찾기 중심이므로, 어른용에선 큰 글씨 단순 목록으로 표시(별도 복잡 UI 없음). 일반은 기존.
- 즐겨찾기 별(추가/삭제) 기능 자체는 두 모드 공통 유지.

## 데이터 흐름

```
useViewMode(localStorage) ──> App 라우트 분기 ──> ElderHome | CitizenHome
                                                     │
        useStops / useFavorites / geolocation ───────┤ (공유)
                                                     ▼
        FacilityBadge · arrivals · walking (공유 원자/라이브러리)
```
버전 전환은 스토어 값만 바꾸고 리렌더. 선택된 정류장·즐겨찾기 등 상태는 스토어/URL에 있어 모드 전환에도 유지.

## 범위 밖 (하지 않음)

- 행정 대시보드(`/admin`)·인쇄 포스터 UI 변경.
- 데이터 파이프라인/시설 매칭 로직 변경(그늘 반경, 도착안내기 소스 추가 등은 별건).
- 폰트 배율만 조절하는 3번째 모드.
- 다국어, 테마(다크모드) 등 미요청 기능.

## 성공 기준

1. 앱 최초 진입 시 어른용(내 정류장 중심) 화면이 뜬다. 즐겨찾기 없으면 최근접 정류장이 큰 카드로.
2. 상단 토글로 어른용↔일반 즉시 전환되고, 새로고침해도 마지막 선택이 유지된다.
3. 데스크톱에서도 폰 폭으로 보인다(웹처럼 퍼지지 않음).
4. 두 모드 모두 3상태·기준일·폴백 규칙을 지키고, `npm test`가 통과한다.
