# 쉼표 정류장 본선(v3) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.
> 설계 근거: `docs/specs/2026-07-16-본선-design.md` (이하 "설계"). 전략: `docs/본선_전략.md`.

**Goal:** 기존 v1+v2 앱 위에 ①comfortScore 코어 ②B2C 편안 경로+정류장 QR ③B2G 쉼표지수(가중치 슬라이더·투트랙·근거카드·예산 시뮬레이터) ④AI 로드뷰 판독 보조 ⑤(P1) 익명 카운터+AI 브리핑을 얹는다.

**Architecture:** stops.json/routes.json 계약 불변. comfortScore·쉼표지수는 프론트 순수 함수 파생 계산. 서버 신설은 P1 카운터(옵트인·실패무시)뿐. AI 판독은 오프라인 보조 도구(사람 캡처→AI 초안→사람 확정→기존 파이프라인).

**Tech Stack:** 기존과 동일 — Vite+React19+TS, Leaflet, zustand, vitest / Python+pandas, pytest. 신규 의존성 없음(P1 카운터 제외).

## Global Constraints

- **정직성 규칙(유지):** 시설 3상태, 근거 없는 `"no"` 생성 금지, "현장 확인" 문구 금지, 수요 "양방향 합산 기준" 병기, 거짓 "실시간" 표기 금지, 노선 단위 수요 주장 금지.
- **합성 점수 규칙(2026-07-16 개정):** 지수는 5조건 하에서만 — ①산식 공개 ②구성요소 실측값 병기 ③출처·기준일 배지 ④**B2G만 숫자 표시, B2C는 숫자 비표시(정렬+이유 문장만)** ⑤미확인의 '없음' 취급 금지(결측은 가중치 재정규화, 지수 명칭은 "개선 검토 우선순위").
- **미확인 ≠ 없음이 산식에도 적용:** comfortScore에서 미확인은 0 가점(감점 아님). 쉼표지수는 가중합(곱셈형 금지 — 결측 정류장이 0점으로 증발하면 안 됨). demand 없는 정류장은 D 항 제외+재정규화+"수요 미확인" 배지.
- **키 보안:** 클라이언트 번들에 LLM 키 절대 금지. 공개 배포는 무키 빌드(`VITE_TAGO_KEY` 비움). 카운터/프록시 엔드포인트 추가 시 CSP `connect-src` 동시 갱신.
- **stops.json / stop.ts 계약 불변.** 스키마 변경이 필요해 보이면 구현하지 말고 총괄에 회부.
- 접근성 하한: 폰트 ≥18px, 터치타깃 ≥48px, 아이콘+한글 병기, WCAG AA. 키보드 없이 핵심 흐름 완결.
- 오프라인 하드 제약: 신규 네트워크 의존(카운터·브리핑)은 전부 실패 무시/폴백. 무한 스피너 금지.
- 커밋: 태스크마다 `git add <명시 경로>`(전체 add 금지). 메시지 끝 `Co-Authored-By: Claude Opus 4.8 (1M context) <noreply@anthropic.com>`.

**공통 타입 (신규, `app/src/types/priority.ts`):**

```ts
export interface PriorityWeights { demand: number; rest: number; routes: number; vuln: number; } // 0~100
export const DEFAULT_WEIGHTS: PriorityWeights = { demand: 25, rest: 25, routes: 25, vuln: 25 };
export interface PriorityRow {
  stop: Stop;
  index: number | null;          // 활성 항 가중합 [0,1]. 계산 불능이면 null(화면 "산정 불가")
  components: {                  // 병기용 실측/정규화 값. 결측 항은 null
    demandMidday: number | null; // 한낮 승차 실측값
    demandQ: number | null;      // 분위수 [0,1]
    restUnknown: number;         // 1 - comfortScore
    routeCount: number; routeQ: number;
    vuln: number | null;         // POI 미확보 시 null(항 비활성)
  };
  missingDemand: boolean;        // "수요 미확인" 배지
}
```
