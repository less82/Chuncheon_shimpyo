## 1. 방향성: B2C + B2G 투트랙

| 트랙 | 직접 공략 배점 |
|---|---|
| B2C 시민앱 | 시민 체감 15 + 서비스성 20 + 시연 일부 |
| B2G 대시보드 | 행정 적용 15 + 분석 타당성 20 + 결합 구조 5 |

한쪽만 하면 ~30점을 구조적으로 버린다. 두 트랙의 연결 고리는 **"시민이 겪는 정보 부족(미확인)을 행정의 조사·설치 업무로 전환하는 흐름"**(§3)이다.

**핵심 수정 2건(초기안에서):**
- **"실시간" 금지** — 승하차 데이터는 2025.6.25~28 나흘치 정적 표본(농촌 223곳 수요 미확인). "실시간"은 TAGO 도착정보에만 사용, 나머지는 "기준일 YYYY.MM" 배지. *"저희는 없는 데이터를 실시간이라고 부르지 않습니다."*
- **단일 합성지수 폐기** — 초기안 "쉼표지수(수요×결핍×취약×노선)"는 미확인이 없음과 동일한 순위 효과를 갖는 산식 모순으로 폐기(Codex 2.1). §3의 2단계 구조로 대체.

## 2. B2C 앱 — "앉아서 기다리는 길"

### 2-1. 타깃: 고령자(65+) 단일 타깃

- 페르소나: *"후평동 78세 김OO 님, 병원 가는 길. 스마트폰은 카카오톡만 쓰심."* — 시민 체감 15점은 "누가, 언제, 얼마나"를 인물로 답해야 한다.
- 타깃을 넓히면 화면이 복잡해져 서비스성이 깎인다. 한 세대에 깊게.

### 2-2. 핵심 기능: "앉아서 기다리는 길" (확인된 시설 우선 추천)

- 정렬 토글: **"가까운 순" / "시설 확인된 곳 우선"(기본)**. 부제: "확인된 시설이 있는 길을 우선 보여드려요".
- 원리: **확인된 시설의 존재(yes)를 우선**하는 추천이며, 미확인을 '시설 없음'으로 단정하지 않는다. (※ "편안함 점수"라는 설명은 화면·발표에서 금지 — 실측 쾌적성이 아니라 확인 시설 개수의 정규화이기 때문. 점수 숫자도 B2C 화면에 비표시.)
- **결과 문구는 시설별로 구체화**(명칭과 결과가 어긋나지 않게 — Codex 재판정 ①):
  - 의자 확인: "앉아서 기다릴 수 있어요"
  - 그늘만 확인: "그늘이 확인된 정류장이에요"
  - 조명만 확인(야간): "조명이 확인된 정류장이에요"
  - 시설 미확인: "시설 정보가 아직 확인되지 않았어요"
- **실효 커버리지를 측정해 공개**: 두 정렬이 다른 결과를 내는 비율, 확인 시설 후보 존재 비율, 추가 도보시간 분포. 낮으면 낮은 대로 공개하고 "조사(§3)가 커버리지를 늘린다"로 연결.

### 2-3. 나머지 B2C 기능

- **정류장 QR 즐겨찾기** — 스캔하면 설치도 가입도 없이 등록(부품 기완성: `?fav=` 딥링크·로컬 QR·A4 포스터). 배포 URL 선고정 필수. 로그인 없음(§4-1). 포스터는 행정 배포물로 B2G와 연결.
- **"지금 기다리기 좋은 곳"** — 폭염 시간대(13~16시)에 그늘 확인 정류장 우선 강조(기존 시설필터 재사용).
- **초대형 글씨 모드 마감** (음성 TTS는 P-거절).
- P1: 도착 알림, `?trip=` 여정 공유.
- **P-거절:** 회원가입/로그인, 챗봇, 전 세대 모드, 실내지도, 원격 동기화.

### 2-4. 화면 구성

```
[홈] 큰 버튼 2개: "🚌 버스 타러 가기" / "⭐ 자주 가는 곳"
[정류장 카드] 도착정보(실시간/폴백 배지) + 시설 4종 3상태·출처 배지 + "이 정류장 QR"
[앉아서 기다리는 길] 토글(가까운 순/시설 확인 우선) + 문장 중심 카드
 ├─ "① 집에서 4분 → ② ㅇㅇ정류장(의자 있음·벤치대장 기준) → ③ 16번 버스 → ④ 병원 앞"
 ├─ 시설별 문구(§2-2) + "가까운 곳보다 1분 더 걷습니다"
 └─ [크게 보기] [자녀에게 보내기(QR)]
[즐겨찾기] 초대형 카드, QR 유입 시 "○○정류장을 즐겨찾기에 넣었어요"
```

**고령자 UX 체크리스트:** 글자 ≥20px·버튼 ≥56px·WCAG AA / 화면당 결정 1개 / 전문용어 금지 / 모든 상태에 문장 폴백 / **타이핑 없이 완결 + 키보드만으로도 조작 가능(둘 다)**.

## 3. B2G 웹 — 2단계 정책 흐름: "조사부터, 그다음 설치"

### 3-1. 구조 (프로젝트의 척추)

```
[현재 데이터: no=0, 미확인 다수]
 1단계 조사 우선순위 = 수요(실측 분위수) + 정보 미확인도   ← 기본 2항
                       (+ 생활지원시설 인접도 — 정책 프리셋에서만 활성)
    "수요가 많은데 정보가 부족한 곳부터 조사한다" — 지금 데이터만으로 성립
    ※ 수요 미확인 정류장은 순위표에 섞지 않고 별도 그룹(순위 없음)
    ※ 표현은 "점수"가 아니라 "조사 검토 순서"
        ↓  로드뷰/현장 조사 (AI 보조 초안 + 사람 확정)
[조사 후 데이터: yes/no 확정 증가]
 2단계 설치 검토 우선순위 = '없음' 확인된 정류장만, 벤치/그늘막/조명 시설별 별도 순위
    ※ 모든 행에 "현장 설치 적합성 미검토" 상태 라벨(보도폭·사유지 등은 데이터 없음 — 아는 척 금지)
        ↓
 시설별 예산 시뮬레이터 → 설치 → 데이터 갱신 (선순환)
```

- **"미확인은 미확인으로"가 문구가 아니라 업무 흐름 자체.** no와 unknown은 변수 수준에서 분리(코드 테스트로 강제).
- 가중치는 **정책 프리셋 3종**(폭염 대응/고령자 이동지원/이용량 중심 — 근거 문구 병기, 인접도는 고령자형에서만 활성) + **프리셋별 순위 안정성 표**(Top10 진입 빈도. "민감도 분석"이 아니라 **"정책 시나리오 비교"**로 지칭 — 과장 금지). 슬라이더는 보조 패널.
- 노선 수는 산식 제외(상관분석 후 재검토), 표시용 원값만.

### 3-2. 공무원의 질문 → 우리의 답

| 질문 | 답 |
|---|---|
| "예산 어디부터?" | 1단계: 조사 검토 순서 → 조사 후 2단계: 시설별 설치 검토 목록 |
| "왜 거기부터 조사하나?" | 근거 카드: 한낮 승차 실측 + 미확인 시설 수 + 산식 공개 |
| "벤치가 필요한 곳과 그늘막이 필요한 곳은?" | 2단계가 시설별 별도 순위 — 조사에서 없음 확인된 곳만 |
| "노인이 많이 쓰는 곳인가?" | 생활지원시설 인접도(병원·경로당·시장 300m — 사람 취약성 측정이 아님을 명시) |
| "보고서에 붙일 수 있나?" | CSV(구성요소 전 컬럼 — 엑셀 재검산 가능) + A4 인쇄 |
| "데이터 오래된 것 아닌가?" | 전 지표 기준일 배지 + 데이터 품질 검증 리포트(하루 제외 안정성·원본 대조 등 7종) |
| "설치 못 하는 곳이면?" | "현장 설치 적합성 미검토" 라벨 고정 — 최종 판단은 행정의 현장 검토 |

### 3-3. 대시보드 산출물

1. **1단계 조사 우선순위** — 표(실측값 병기, 순위가 주인공)·지도·CSV. 수요 미확인 그룹은 하단 별도 섹션.
2. **2단계 설치 검토** — 시설별 서브탭(벤치/그늘막/조명). 조사 반영 전엔 "1단계를 먼저" 안내.
3. **근거 카드** — 산식 전체 공개 + 항별 (실측 → 정규화 × 가중치) 분해 + 출처·기준일 배지 + 시간대 그래프("양방향 합산" 병기).
4. **시설별 예산 시뮬레이터** — 트랙별 단가("가정, 편집 가능" 라벨) → 대상 정류장 N곳 + **"표본기간(2025.6, 4일) 한낮 승차 합계 M건"** ("수혜"·"일평균" 표현 금지).
5. **데이터 품질 검증 리포트**(`docs/데이터_검증.md`) — 분석 타당성 20점의 실탄: 하루 제외 Top20 안정성, 동명 조인 충돌, 결측 223곳 분포, 원본 20곳 대조, 수요-노선 상관, 기준일 불일치표.

### 3-4. 확보할 공공데이터

| 데이터 | 용도 | 비고 |
|---|---|---|
| 병원·경로당·전통시장 POI | 생활지원시설 인접도 | 기보유 지오코딩 재사용, 미확보 시 항 자동 비활성 |
| 행정동별 고령인구(KOSIS) | 인접도 보조·발표 근거 | 정적 CSV 1개 |
| 주최측 요청 5건 / 본선 OT 재브리핑 데이터 | 현장 결합 라이브(§7) | 도착·배포 시 판단 |

## 4. 데이터 아키텍처 — "개인 데이터는 기기에"

### 4-1. "개인정보 미수집"이 정체성인 이유

① 고령자에게 가입은 최대 이탈 장벽 ② 서버 없는 정적 PWA와 한 몸 ③ 개인정보보호법 책임 회피(고령자 이동패턴은 민감 정보) ④ 행정 채택 장벽 제거 ⑤ 심사 단골 질문 원천 차단. (보안점검 문서에 "개인정보 미수집 — 통과" 검증됨)

### 4-2. 계층 설계

```
Tier 1  개인 데이터   → localStorage (즐겨찾기 id·설정. 기기 소유, 오프라인 완전 동작)
Tier 2  사람→사람 공유 → URL/QR 페이로드 (?fav= 구현 완료, ?trip= 확장 — 서버 없이 기기→기기)
Tier 3  익명 집계     → 【P2 실험 — 본선 범위 밖】 클라우드 카운트({stop_id, event, hour, date}만).
        착수 조건: 본선 필수 작업 완료 + 팀 합의. 정직 표기: "클라이언트는 식별자를 보내지 않으나
        호스팅 인프라 접속 로그에 IP가 남을 수 있어 로그 보존을 최소화" — "IP 미저장"을 서버까지
        보장하는 것처럼 말하지 않는다. 미착수 시 로드맵 슬라이드로만.
Tier 4  원격 동기화   → 하지 않음 (QR 대리등록으로 시나리오 성립)
```

- "자녀가 저장한 즐겨찾기 가져오기"는 **로컬로 이미 된다**(URL/QR에 데이터가 실림). 안 되는 건 원격 실시간 동기화뿐(의도적 제외).
- AI의 자리: **집계 위에서만**(어드민 브리핑 — P2, 사전 생성 정적 JSON, 클라이언트 LLM 키 금지). B2C 추천은 온디바이스 규칙 기반 — 개인 데이터에 LLM을 붙이지 않는다.

## 5. 배점 → 행동 매핑

| 배점 | 대응 | 발표에서 보여줄 것 |
|---|---|---|
| 행정 적용 15 | 2단계 정책 흐름 + 근거카드 + 인쇄물/QR 포스터 | A4·포스터 실물 배부 |
| 시민 체감 15 | 고령자 페르소나 + 앉아서 기다리는 길 + 가입 없는 QR + **사용자 테스트 결과** | "1분 더 걷고, 앉아서 기다립니다" + 개선 전후 1장 |
| 분석 타당성 20 | 산식 공개 + no/unknown 변수 분리 + **데이터 품질 검증 리포트** | 검증 리포트 발췌 1장, "미확인≠없음이 산식에서 지켜지는 방법" |
| 시연 가능성 20 | 라이브 데모 + 오프라인 폴백("최초 1회 접속 후" 조건 병기) | 와이파이 끄고 데모 계속 + **조사 CSV 투입 전후 라이브** |
| 서비스성 20 | 고령자 UX + 김OO 님 시나리오 완주 | 탭 순서 그대로 시연 |
| 데이터 적절성 5 | 춘천시 CSV 6종 + POI·고령인구 | 출처·기준일 슬라이드 |
| 결합 타당성 5 | ID/공간/이름 3중 조인 + 조사 데이터 환류 구조 | 파이프라인 다이어그램 1장 |

## 6. 작업 우선순위 (타임라인 2구간)

- **본선 전(7/17~7/30):** 2단계 엔진+대시보드, 앉아서 기다리는 길(토글·시설별 문구·커버리지 측정), 정류장 QR, 데이터 품질 검증 리포트, 로드뷰 조사(AI 보조·효율 측정·컷라인 50곳), POI 인접도, 고령 사용자 3~5명 테스트, 접근성(키보드 포함), 오프라인 E2E 매트릭스, 배포 URL 고정+포스터
- **본선 현장(7/31~8/1):** 신규 기능 금지. 주최측 재브리핑 데이터 현장 결합 + 14분 초단위 리허설 ≥3회 + 네트워크 3조건 검증 + 공격 질문 15문 답변 카드
- **P2 실험(조건부):** 익명 카운터·AI 브리핑(§4-2 Tier 3 조건 충족 시에만)
- 태스크·담당·컷라인: `docs/superpowers/plans/2026-07-16-본선.md`

## 7. 발표 14분 구성(안)

1. (2분) 문제 — "공공데이터 결합으로 그늘 확인 4곳, **1,886곳은 없음이 아니라 미확인**입니다"(단독 "그늘 4곳뿐" 금지) + 김OO 님 이야기
2. (3분30초) B2C 데모 — QR 스캔 등록(가입 없음) → 앉아서 기다리는 길(토글·시설별 문구) → 와이파이 오프 → 계속 동작("최초 1회 접속 후 오프라인 동작입니다")
3. (4분30초) B2G 데모 — 1단계 조사 검토 순서 → **조사 확정본 투입 라이브**: *"본선 전에 사람이 검수한 로드뷰 조사 결과를 행정 조사 입력 형식으로 투입하면, 설치 검토 목록이 자동 생성됩니다"* — 전후 수치 4종 표시(투입 전 no=0·후보 0곳 / 조사 N곳: yes A·no B·unknown C / 투입 후 시설별 후보 수 / 제외된 unknown 수) → 2단계 시설별 순위 → 예산 → A4·포스터 배부
4. (2분) 데이터 — 6종 결합 구조 + 검증 리포트 발췌 + "개인 데이터는 폰 밖으로 나가지 않습니다"
5. (2분) 확장 — 조사→설치→갱신 선순환, 타 지자체 복제. 시민 신호는 **10~15초 한 문장만**: *"향후에는 식별정보 없는 시민 반응 집계를 보조 신호로 추가할 수 있지만, 현재 우선순위에는 검증되지 않은 시민 신호를 섞지 않았습니다."*
