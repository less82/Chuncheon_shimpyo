# 쉼표 정류장 — 대상 전략

> 제품 정의는 `docs/제품_기획.md`가 유일한 기준이다. 이 문서는 심사와 발표 전략만 다룬다.

## 1. 승부할 한 문장

**정류장 QR이 출발지를 알고, 이용자는 목적지만 말하면 된다.**

BIT가 없는 정류장에 ‘QR 음성 도착안내기’를 설치한다는 기획이 아니다. QR은 앱 진입과 정류장 식별, 앱의 음성은 목적지 입력, 공공교통 데이터는 노선·도착정보 제공을 담당한다.

## 2. 왜 이 접근이 유리한가

| 심사 관점 | 보여줄 증거 |
|---|---|
| 지역 문제 | 춘천 현행 정류장·노선·BIT 자료로 정보 접근 공백을 특정 |
| 시민 체감 | 고령 사용자가 타이핑 없이 목적지 버스와 도착시간을 확인하는 실기기 시연 |
| 데이터 활용 | 관리번호↔4자리 번호↔노선 정차순서↔TAGO 도착정보의 명시적 연결 |
| 기술 구현 | QR 딥링크, 음성 인식, 경로 계산, 실시간/폴백 상태 처리 |
| 행정 적용 | 정류장별 QR 생성·배포·점검 대장과 단계적 적용안 |

기존의 시설 조사·설치 우선순위는 보조 트랙이다. 발표의 첫 문제와 첫 데모에 올리지 않는다.

## 3. 발표 흐름

1. 고령 이용자가 정류장에서 목적지 버스와 도착시간을 찾기 어려운 장면
2. QR 촬영만으로 현재 정류장이 정확히 잡히는 장면
3. 목적지를 말하고 실제 운행 버스·방면·도착예정시간을 확인하는 장면
4. 데이터 연결 구조와 잘못된 추정을 막는 규칙
5. 고령 사용자 테스트의 성공률·시간·오조작 전후 결과
6. BIT 미설치 정류장부터 QR을 배포하고 관리하는 행정 적용안
7. 시설정보·즐겨찾기·보호자 공유는 확장 화면으로 짧게 제시

## 4. 반드시 확보할 실탄

- 정류장 QR 5~10곳의 현장 대응 정확도
- 각 QR에서 안내되는 대표 목적지 노선의 정차순서·방면 수기 검산
- TAGO 도착예정시간과 실제 앱 표시 비교 기록
- 65세 이상 5명 이상의 비보조 태스크 결과
- BIT 설치/미설치 구분과 QR 적용 후보 수
- QR 제작·부착·점검에 필요한 예상 업무와 비용

한낮 승차량은 개별 정류장 ID 대응이 해결되기 전까지 이 목록의 우선순위를 정하는 데 쓰지 않는다.

## 5. 공격 질문에 대한 기준 답변

| 질문 | 답변 |
|---|---|
| QR이 음성안내를 하나요? | 아니요. QR은 앱을 열고 정류장을 식별합니다. 음성은 앱의 목적지 입력 방식입니다. |
| BIT를 대체하나요? | 대체가 아니라 미설치·이용곤란 지점의 저비용 보완 수단입니다. |
| 어르신이 QR을 쓸 수 있나요? | 가정하지 않고 실제 사용자의 완료율·시간·도움 여부로 검증합니다. |
| 도착정보가 틀리면요? | 실시간 출처·생성시각을 표시하고, 없으면 실시간이라 부르지 않습니다. |
| 왜 이 정류장부터 설치하나요? | 현행 운영 정류장, BIT 공백, 현장 수요 검증을 분리해 근거를 공개합니다. 불명확한 승차 ID는 쓰지 않습니다. |

## 6. 중단 조건

- QR ID와 현행 정류장의 1:1 대응이 확인되지 않음
- 노선 정차순서나 방면을 추정으로 안내함
- 실시간이 아닌 값을 실시간 도착정보로 표시함
- 고령 사용자 테스트 없이 ‘고령자에게 편리하다’고 발표함
- 시설정보나 대시보드가 핵심 사용자 흐름보다 발표 비중이 커짐
