# 쉼표 정류장 앱 스토리보드

> 적용 대상은 이미 앱을 알고 반복 이용하는 시민이다. 공통 QR의 첫 방문 안내와 분리한다. 화면은 320×461px 실제 사용영역을 기준으로 하며 세로 스크롤 없이 핵심 행동을 완료해야 한다.

## 정보 우선순위

1. 지금 탈 버스의 실제 도착 예정시간
2. 탈 정류장명·4자리 번호·목적지 방면
3. 출발 위치에서 탈 정류장까지 도보시간
4. 실시간 정보의 제공 여부
5. 반복 이용을 위한 즐겨찾기
6. 해당 승차 정류장의 상태 알리기

배차간격은 도착 예정시간이 아니다. TAGO 실시간 값을 받지 못하면 큰 시간 자리에 배차간격을 넣지 않고 `실시간 도착정보를 불러오지 못했어요`라고 표시한다. 계산할 근거가 없는 총 소요시간과 목적지 도착예상도 만들지 않는다.

## 화면 흐름

| 장면 | 화면 목적 | 화면에 크게 보일 정보 | 사용자 행동 | 다음 상태 |
|---:|---|---|---|---|
| 1 | 반복 이용 시작 | `버스`, `정류장`, 저장한 출발–목적지와 도착정보 | 즐겨찾기 또는 `버스` 선택 | 저장 경로 결과 또는 장소 입력 |
| 2 | 입력 방법 선택 | `어디서 출발하세요?`, `어디로 가세요?`, 말하기 버튼 | 음성 입력 또는 `지금은 말할 수 없어요` | 음성 확인 또는 직접 입력 |
| 3 | 출발 위치 확인 | 검색한 장소의 지도 핀·주소 | `이 위치에서 출발` | 목적지 입력·확인 |
| 4 | 경로 계산 | `탈 버스와 승차 정류장을 찾고 있어요` | 기다림 | 올바른 방향의 승차 정류장 자동 산출 |
| 5 | 버스 도착정보 | 도착시간 → 버스번호 → 승차 정류장·방면 → 도보시간 | 버스 정보 확인 | 즐겨찾기 또는 정류장 상태 알리기 |

## 장면별 상태

### 1. 앱 메인

- 첫 행동은 `버스`와 `정류장` 두 개다.
- 반복 이용자는 즐겨찾기 카드 전체를 눌러 바로 도착정보로 이동한다.
- 설명문·`보기`·장식 아이콘을 넣지 않는다.
- 즐겨찾기 카드에는 `출발 → 목적지`, 방면, 실시간 도착정보만 둔다.

### 2. 출발·목적지 입력

- 음성 모드에서는 `출발지 말하기`, `목적지 말하기`를 제공한다.
- 직접 입력은 사용자가 버튼을 눌렀을 때만 열린다.
- 타이핑만으로 다음 단계로 이동하지 않는다.
- `지도에서 찾기`를 누른 뒤에만 장소 검색 API를 호출한다.

### 3. 위치 확인

- 지도 핀과 주소가 없으면 `확인`이라는 표현을 쓰지 않는다.
- 검색 실패 시 입력 화면을 유지하고 장소명·주소 재입력을 안내한다.
- 사용자가 `이 위치에서 출발`을 눌러야 좌표를 확정한다.

### 4. 계산

- 주변 정류장 목록을 사용자가 고르게 하지 않는다.
- 출발 좌표와 목적지로 올바른 방향의 승차 지점을 시스템이 결정한다.
- 경로를 확정할 수 없으면 임의 정류장이나 맞은편 정류장을 붙이지 않는다.

### 5. 결과

- 첫 카드 한 장만 보여준다.
- 실시간 도착은 최대 두 대만 큰 글자로 표시한다.
- 즐겨찾기는 결과 아래의 짧은 보조 행동으로 둔다.
- `정류장 상태 알리기`는 산출된 승차 정류장을 다음 화면에 그대로 전달한다.

## 예외 화면 문구

| 상황 | 문구 | 금지 |
|---|---|---|
| 장소 검색 실패 | `춘천시에서 위치를 찾지 못했어요` | 비슷한 이름을 임의 확정 |
| 경로 없음 | `이 위치 주변에서 목적지로 가는 버스를 찾지 못했어요` | 시설·도보 경로 추천으로 대체 |
| 실시간 도착 실패 | `실시간 도착정보를 불러오지 못했어요` | 배차간격을 도착예정처럼 표시 |
| 마이크 사용 불가 | 원인별 오류와 `직접 쓰기` | 네트워크 전체 장애로 표현 |

## 구현 경계

- 현재 구현: 장소 검색 지도, 출발 좌표 주변 승차 정류장 산출, 직행 노선 탐색, TAGO 도착 조회, 즐겨찾기, 상태 알리기 연결
- 미구현: 외부 대중교통 경로 API 기반 총 소요시간·목적지 도착예상·환승 최적화
- 미구현 값을 데모용 추정치로 채우지 않는다.
