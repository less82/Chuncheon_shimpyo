## 1. 목적 · 완료 기준

**완료 = 본선 발표 데모 시나리오가 행사장 네트워크 불안 상태에서 통과:**
1. (B2C) 정류장 QR 스캔 → 가입 없이 즐겨찾기 자동 등록 + 확인 문구
2. (B2C) 목적지 선택 → **편안 경로**: 후보가 편안한 순으로 정렬, 점수 숫자 없이 이유 문장 표시
3. (B2C) 와이파이 차단 후에도 위 흐름 계속 동작(폴백 문구로 전환)
4. (B2G) 쉼표지수 우선순위 테이블 + **가중치 슬라이더** 조작 → 순위 실시간 재정렬
5. (B2G) 행 클릭 → 근거 카드(산식 공개·구성요소 실측값·출처 배지) → CSV/A4 출력
6. (B2G) 예산 슬라이더 → 커버 정류장 수·수혜 승차 건수 재계산
7. (B2G) 조사 필요 목록(투트랙) 표시 — "미확인은 미확인으로"

## 2. 규칙 개정 (2026-07-16 합의 — Notion "본선 팀 합의 필요 사항" 페이지 결정 ①)

기존: "대시보드 합성 점수 표시 금지, 조건 필터만" → **개정: "설명 불가능한 합성 점수 금지"**

합성 점수(지수)는 아래 5조건을 전부 만족할 때만 허용:
1. **산식 전체 공개** — 근거 카드에서 계산 과정 열람 가능
2. **구성 요소별 실측값 병기** — 순위 옆에 승차량·시설 3상태·노선 수 원값 표시
3. **출처·기준일 배지 동반** — 모든 지표에 "YYYY.MM 기준"
4. **B2G에서만 숫자 표시, B2C는 숫자 비표시** — 시민앱은 정렬 결과+이유 문장만
5. **미확인을 '없음'으로 취급하는 산식 금지** — 결측 항은 가중치 재정규화로 처리(§4), 지수 명칭은 "점수"가 아니라 **"개선 검토 우선순위"**

나머지 v1 정직성 규칙은 전부 유지: 시설 3상태(근거 없는 "no" 금지), "현장 확인" 문구 금지, 수요 "양방향 합산 기준" 병기, 거짓 "실시간" 표기 금지, 노선 단위 수요 주장 금지.

## 3. 핵심 구조 — comfortScore 코어 공유

```
comfortScore(stop) ∈ [0,1]        ← 시설 '있음 확인' 기반. 정류장 자체 속성. 단일 모듈.
  ├─ B2C 편안 경로 = comfortScore − 도보거리 패널티   (개인 위치. 숫자 비표시, 이유 문장만)
  └─ B2G 쉼표지수 = 수요·미확인도(=1−comfort)·노선수·취약의 가중합  (수요·정책 가중. 숫자는 보조)
```

- 도보 패널티는 편안 경로에만, 수요는 쉼표지수에만. 경계를 섞지 않는다.
- 두 지수가 같은 시설 판정을 공유 → 시민앱/대시보드 간 모순 구조적으로 불가.
- 발표 문장: "시민에게 편안한 정도를 뒤집으면, 행정이 투자할 순서가 됩니다."

### 3.1 comfortScore 정의

```ts
// 확인된 시설만 가점. 미확인은 0점이지 감점이 아님(없음 취급 금지).
comfortScore(stop, opts?: {night?: boolean}) =
  (seat==="yes" ? W_SEAT : 0) + (shade==="yes" ? W_SHADE : 0)
  + (night && light==="yes" ? W_LIGHT : 0)   // 야간에만 조명 반영
  / 활성 가중치 합                             // 0~1 정규화
기본 가중치: W_SEAT=W_SHADE=W_LIGHT=1 (동일)
```

## 4. B2G 쉼표지수 (개선 검토 우선순위)

**가중 선형 결합(WLC — 다기준 의사결정 표준 기법):**

```
index(stop, w) = ( w.demand·D + w.rest·U + w.routes·R + w.vuln·V ) / Σ(활성 w)

D = 한낮(11~16시) 승차량의 분위수 정규화 [0,1]      ← demand 없으면 결측
U = 1 − comfortScore(stop)                        ← "쉼 미확인도" (없음 단정 아님)
R = 노선 수 분위수 정규화 [0,1]
V = 취약 가중치: 병원·경로당·시장 300m 인접 [0,1]   ← POI 미확보 시 항 자체 비활성
```

- **정규화는 분위수**(min-max 아님) — 터미널급 극단값이 저울을 왜곡하지 못하게.
- **결측 처리(정직성 핵심)**: demand 없는 정류장(농촌 223곳 포함)은 D 항을 빼고 남은 가중치로 재정규화 + 행에 "수요 미확인" 배지. 곱셈형 금지(결측=0점 증발은 미확인의 없음 취급).
- **가중치 슬라이더**: `w` 4개(0~100, 기본 동일). 조작 시 실시간 재정렬. "가중치는 정책 판단 — 담당자가 정한다"가 심사 방어 논리. 슬라이더를 움직여도 상위권이 유지되면 강건성 데모.
- **투트랙 출력**:
  - **개선 검토 우선순위** — 수요 실측 있는 정류장 대상 랭킹 (설치 검토용)
  - **조사 우선순위** — 시설 미확인 & (수요 상위 or 수요 미확인) 정류장 목록 (로드뷰/현장조사용, §7 AI 판독의 입력)

## 5. B2C 편안 경로

- `planTrip` 결과의 출발 후보 정류장들을 `comfortScore − walkPenalty`로 정렬 (기존 로직 위에 정렬 기준만 추가).
- **화면에는 점수 없음.** 카드마다 이유 문장 자동 생성: "의자·그늘 있음 — 가장 가까운 곳보다 1분 더 걷지만 앉아서 기다립니다" / 시설 미확인이면 그 사실을 그대로("시설 정보가 확인되지 않은 정류장입니다").
- **"지금 기다리기 좋은 곳"**: 폭염 시간대(13~16시)면 그늘 확인 정류장 우선 강조(기존 시설필터 재사용).
- **정류장 QR**: 카드에 "이 정류장 QR" 버튼(`buildShareUrl([stop.id])` 재사용) + PrintPoster에 QR 삽입 + `?fav=` 유입 시 확인 문구. 로그인·계정 없음.

## 7. AI 로드뷰 판독 (조사 인력 절감 — 사람 캡처 → AI 판독 → 사람 확정)

1. 파이프라인이 조사 우선순위 상위 150곳 목록 생성 (§4 투트랙)
2. 팀원이 카카오맵 로드뷰 **스크린샷만** 캡처(판단 없음, 1곳 ~30초). 파일명 = 관리번호
3. 멀티모달 AI(Claude)가 배치 판독: 의자/지붕(쉘터)/조명 → 있음/없음/판별불가 + 근거 한 줄 → `roadview_survey_template.csv` 형식 초안 자동 작성
4. 팀원이 초안 검수·확정(판별불가 위주). **사람이 최종 확정하므로 배지는 기존 규칙 그대로** "로드뷰 확인 (촬영 YYYY.MM)"
5. `build_stops.py` 재실행 — 로드뷰 우선 반영은 기존 구현, 코드 수정 없음

- 로드뷰 이미지의 프로그램적 수집(크롤링)은 지도 서비스 약관 위반 소지 → **사람 캡처가 원칙**.
- 발표 활용: "AI로 조사 비용 1/5, 시가 같은 방식으로 나머지를 채울 수 있다"(확장성).

## 8. Tier 3 익명 카운터 + AI 브리핑 (P1)

- **카운터**: 즐겨찾기 등록/QR 스캔 시 `{stop_id, event, hour, date}` 카운트만 전송(식별자·IP 저장 금지). 옵트인, 실패 시 조용히 무시(오프라인 원칙). 백엔드는 Supabase 테이블 1개 또는 Cloudflare Worker+KV — 팀 계정 사정으로 선택.
- **시민 신호**: 어드민에 즐겨찾기 카운트 순위/히트맵. 쉼표지수의 보조 지표(별도 표시, 산식 합성 안 함 — 표본 적어 왜곡 위험).
- **AI 브리핑**: 집계 통계 → 자연어 요약. **클라이언트 번들에 LLM 키 절대 금지**(Vite `VITE_` 인라인 실측 확인됨). 데모는 사전 생성 정적 JSON, 라이브는 서버리스 프록시만.
- 도입 시 필수 후속 3건: CSP `connect-src`에 엔드포인트 추가 / "서버 0개" 문구를 "개인정보 서버 0"으로 / 보안점검.md 외부 전송 목록 재검증.

- **공개 배포는 무키 빌드** — `.env`의 `VITE_TAGO_KEY` 비우고 빌드(도착정보는 배차 폴백 자동). 로컬 dist에 키 인라인 사고 이력 있음(7/16 발견·삭제).
