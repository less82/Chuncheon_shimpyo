# 쉼표 정류장 — 시설 변화감지 AI 설계 (2026-07-22)

> 브레인스토밍 합의 기록. 시민 민원 사진에서 정류장 시설(벤치·지붕/차양·도착안내기)의
> **유무를 판독**하고, 현재 기록과 **대조해 불일치를 담당자에게 회부**하는 행정 보조 도구.
> 계약: `app/src/types/stop.ts`, `docs/데이터_AI_계약.md`. 정직성 규칙: `docs/데이터_현황.md`.

## 1. 의도 (왜 만드는가)

- 정류장 1890곳 중 시설 대부분이 **미확인**(그늘 1886 / 의자 1065 / 조명 1082 / 도착안내기 1554 미확인).
- 이 제품의 정체성은 "근거 없는 '없음' 금지 = 정직"이다. 그러나 **정직성만으로는 데이터가 텅 빈다.**
  미확인 천지면 시민앱의 핵심 가치("벤치·그늘 있는 정류장 찾기")가 작동하지 않는다.
- 그래서 이 AI의 진짜 할 일: **미확인·노후 데이터를 사람이 확정하는 비용을 낮추는 행정 가속기.**
  = 정직성을 **규모 있게** 유지하는 다리.

### 사용자 구분
- **B2G(행정)** 도구다. 공무원 검수 큐가 본체. 시민앱에는 **확정된 결과만** 노출한다.
- 시민은 이 모델을 직접 만지지 않는다(제보 사진을 올릴 뿐).

## 2. 간판 가치 = 변화 중 "없어짐/고장" 감지 (신설 제외)

첫 원칙에서 도출한 비대칭:

- **신설(추가)은 행정이 이미 안다** — 예산·공사 기록이라는 종이 흔적이 남는다. AI 잉여.
  (대장엔 있는데 앱 데이터에 미반영인 경우는 **AI가 아니라 대장 조인 재실행**으로 해결.)
- **없어짐/파손/고장은 종이 흔적이 없다** — 담당 부서도, 예산도, 갱신 트리거도 없다.
  대장 신뢰도가 시간에 따라 새는 건 거의 전부 이 방향. **여기가 시민 사진의 값어치.**

### 트리거 = 시민 제보 (선제 전수조사 아님)
- 시민이 불편해서 제보한다 = "원래 없었거나(미확인)" 또는 "있다 없어졌거나". 제보 하나로 둘이 통합된다.
- **제보가 들어온 정류장만** 처리하므로 1890곳을 미리 훑을 필요가 없다. (초기 전수적재는 이 문서 범위 밖 — 뒤 §9.)

## 3. 범위 (In / Out)

**v1 (이 문서):**
- ✅ 3클래스 **유무 판독 + 위치 박스**: `지붕/차양`, `벤치`, `BIT 전광판`.
- ✅ 현재 기록과 대조한 **불일치 회부**(discrepancy) 로직.
- ✅ 관리자 대시보드 **검수 큐** + 사람 확정 → 반영.
- ✅ 정직성 게이트(§5) + 감사로그.

**v1에서 하지 않음:**
- ❌ **파손 여부 자동 판단.** 데이터 희소성으로 신뢰 불가 → **사람이 판단**(AI는 박스 크롭까지만).
- ❌ 신설(추가) 자동 반영 — 대장 조인 담당.
- ❌ 조명(light) 판독 — 주간 사진으로 야간 점등 확인 불가(기존 규칙 유지).
- ❌ 시민앱 내 AI UI.

**v2 로드맵(데이터 축적 후):**
- 운영 중 파손 사례가 쌓이면 **파손 분류기**를 추가. "쓸수록 자라는 기능."

## 4. 아키텍처

```
[제보 사진(정류장ID 첨부)]        [탐지]                 [불일치 판정]           [사람]            [반영]
 시민 민원 사진        →  YOLO 로컬추론(Roboflow  →  change_detect.py     →  검수 큐(대시보드) →  apply_roadview
 (+선택: 로드뷰 캡처)      export, pipeline/)         현재기록 대조·우선순위    확정/반려/수정      → build_stops → stops.json
                          3클래스 {label,conf,bbox}                          + 감사로그
```

- **추론은 `pipeline/`에서 로컬 배치.** Roboflow에서 YOLO 가중치를 export해 로컬 실행 →
  호출당 과금 0, 오프라인, 계약이 요구하는 "별도 검증셋 정확도 측정" 가능.
- **기존 초안→검수→반영 패턴 계승.** AI는 `_draft`만 만들고, 사람 검수 후 `roadview.apply_roadview`로 반영.
  새 정직성 규칙을 만들지 않고 `roadview_ai_draft.py`의 가드를 일반화해 재사용.
- **시민 사진**은 계약 `citizen_reports`에 연결 — 정류장ID가 첨부돼 오므로 지오코딩 불필요.
- **BIT 클래스**의 값어치 = 탐지가 아니라 **BIT 대장(345단말) 노후 교차검증**(대장=있음인데 사진=없음 → 철거 의심).

## 5. 정직성 게이트 (불일치 판정 규칙)

작업 단위는 "탐지"가 아니라 **"불일치(discrepancy)"** 다.

| 현재 기록 | 사진 판독 | 판정 | 처리 | 우선순위 |
|---|---|---|---|---|
| 있음 | 탐지됨 | 일치(갱신) | 관측일만 갱신 | — |
| **있음** | **미탐지(고신뢰)** | 없어짐 의심 | 공무원 확인 큐 | 🔴 |
| 미확인 | 탐지됨 | 신규 제안(있음) | 공무원 확정 | 🟡 |
| 미확인 | 미탐지 | **미확인 유지** | 무액션(절대 자동 '없음' ❌) | — |
| 없음 | 탐지됨 | 재설치 의심 | 공무원 확인 큐 | 🔴 |
| BIT대장=있음 | 미탐지 | 대장 노후 의심 | 교차검증 플래그 | 🔴 |

절대 규칙:
1. **미탐지 → 절대 자동 '없음' 아님.** 오직 공무원만 status='no' 확정(source='roadview').
2. 모든 AI 산출은 **미확정.** 공무원 확정 전 stops.json 미반영.
3. **신뢰도 임계값** 미달 → 제안 안 함(잡음 억제). 임계값은 검증셋으로 조정.
4. 모델은 완벽할 필요 없음 — 제안만 하고 사람이 확정하므로, 오차는 데이터 오류가 아니라 검수 품으로만 전가.
5. 감사로그: 누가/언제/무엇을(모델 버전·신뢰도 포함) 바꿨는지 기록.

## 6. 모듈

| 모듈 | 책임 | 의존 |
|---|---|---|
| `pipeline/facility_vision.py` | YOLO export 로드, 사진 1장 → `[{class, conf, bbox}]`. 순수 추론, 쓰기 없음. | YOLO 가중치 파일 |
| `pipeline/change_detect.py` | (현재기록 + 판독) → 불일치 레코드. **간판 로직.** §5 표 그대로. | facility_vision, stops |

| 반영 | 확정분 → `roadview.apply_roadview`(재사용) → `build_stops`. | 기존 |

## 7. 데이터 · 학습 계획

- **클래스 3종 + 라벨 규칙(Roboflow 바운딩박스):**
  - `지붕/차양`: 정류장 상부 지붕·차양·접이식 그늘막 등 **구조물**. **나무 그늘은 제외**(구조물만).
  - `벤치`: 대기용 벤치/의자.
  - `BIT`: 버스도착정보 안내 전광판/단말(LED/LCD).
  - 규칙: 인스턴스마다 박스 1개. 50% 이상 가려지면 스킵. 다양한 각도·거리로 수집.
- **부트스트랩:** 현재 `data/춘천시 정류장 사진`(64장) + 로드뷰 캡처 추가. **64장은 시작점** —
  운영 정확도엔 **클래스당 수백 장** 라벨링 스프린트 필요(정직하게 명시).
- **학습·export:** Roboflow에서 YOLOv8 학습 → **가중치 export**(로컬 추론용). 호스팅 API 미사용.
- **검증:** 학습에 안 쓴 **별도 검증셋**으로 클래스별 정확도(mAP) 측정. 데모 몇 건을 전체 정확도처럼 발표 금지(계약).

## 8. 성과 지표

- **행정효율**: 공무원 시간당 확정 정류장 수(핵심).
- **데이터 신선도**: 시설 관측일 평균 경과.
- **잡아낸 불일치 수**: 없어짐/노후 회부 건수 및 확정 비율.
- 모델 mAP(클래스별) — 보조 지표.

## 9. 리스크 · 중단 조건

- ⚠️ **본선 계획 대조 필수.** 이건 `docs/superpowers/plans/2026-07-16-본선.md`의 🛑 중단 조건에 없던 **신규 범위**.
  데모까지 시간이 빠듯하면 **본선 후 로드맵**으로 두고, 본선에선 "설계 + PoC"까지만 보여주는 안전안 검토.
- **데이터 부족 리스크**: 64장으로는 데모급. 운영 정확도는 라벨링 스프린트 성공에 종속.
- **채택 리스크**: 춘천시가 검수 인력을 실제 배치하는지에 운영 가치가 종속. 발표에서 과장 금지.
- **정직성 회귀 금지**: 어떤 경로로도 AI가 status='no'를 자동 생성하지 않는지 테스트로 고정.
