## 시민 장면

### C1. 진입

```text
안녕하세요. 가까운 정류장을 찾아드릴게요.
[현재 위치로 찾기]
[정류장 직접 찾기]
```

하단에 위치가 서버에 장기 저장되지 않는다는 짧은 설명을 둔다.

### C2. 대상 정류장 확인

```text
가까운 정류장 2곳

[석사극동아파트 · 2122]
 강원대학교 방면 · 도보 2분

[석사극동아파트 · 2123]
 동내면 방면 · 도보 3분
```

가장 가까운 후보를 자동 확정하지 않는다. 방면·4자리 번호·랜드마크를 함께 보여준다.

### C3. 도착·대기환경

```text
석사극동아파트 2122 · 강원대학교 방면

300번  7분 후   | 다음 18분 후
실시간 · 14:21 생성

의자 있음      그늘 미확인
쉘터 없음 확인  조명 있음
버스 도착 안내기 있음
[근거와 확인일]

[이 정류장에서 불편한 점 말하기]
```

도착 API 실패 시 도착 카드만 실패상태로 바꾸고 시설·제보 기능은 유지한다.

### C4. 불편 입력

```text
불편한 점을 말씀해주세요.
[말하기 시작]

또는
[앉을 곳] [햇빛·비] [어두움]
[도착 안내 화면] [시설정보가 다름]
```

TTS가 끝난 뒤 STT를 시작하고, 음성 실패 시 버튼과 글자입력을 제공한다.

### C5. AI 구조화 확인

```text
말씀하신 내용을 이렇게 이해했어요.

대상: 석사극동아파트 2122
시설: 버스 도착 안내 화면
문제: 작동하지 않음
지속성: 며칠째

[맞아요, 제출] [내용 수정]
```

AI가 틀린 상태로 자동 제출하지 않는다.

### C6. 완료

```text
알려주셔서 감사합니다.
정류장 확인과 개선 검토 자료로 활용됩니다.

[현재 확인 상태 보기]
[공식 답변이 필요하면 춘천시 민원 채널]
```

## 행정 장면

### A1. 신규 신호

| 상태 | 정류장 | AI 분류 | 반복 | 공식자료 |
|---|---|---|---:|---|
| 검토 필요 | 2122 석사극동아파트 | BIT 작동 불량 | 4건 | BIT 설치 |

제보 수만으로 순위를 만들지 않는다. 반복, 공식자료 충돌, 최근성, 미확인을 분리해서 보여준다.

### A2. AI 근거 카드

```text
권고: 유지보수 현장확인

왜 확인해야 하나요?
- 공식 BIT 대장: 설치(2025-11-14)
- 시민 발화: 작동 불량 4건(최근 7일)
- AI 해석: 설치 여부가 아니라 작동상태 충돌
- 불확실성: 현장 확인 전

[원문 보기] [승인] [반려] [추가자료 요청]
```

### A3. 현장 확인

```text
AI 권고: BIT 작동 불량 가능성
담당자 결정: 현장조사 배정
조사 결과: 전원 이상 · 수리 요청
확정자/시각/근거 첨부
```

### A4. 시민 환류

조사 전 `BIT 있음`을 단순 유지하지 않고 `점검 중`과 확인일을 시민 화면에 반영한다. 수리 완료 후 사람이 `정상 작동`을 확정하면 다시 갱신한다.
