# 쉼표 정류장 — 사용자 흐름

> 기준: `docs/제품_기획.md`. 이 문서는 화면 순서와 예외만 정의한다.

## 1. 정류장 현장의 고령 이용자

```text
1. 정류장에 부착된 QR을 기본 카메라로 촬영한다.
2. 가입·설치 없이 `/qr_main?from={관리번호}`가 열린다.
3. “현재 출발 정류장”의 정류장명·4자리 번호·방면을 확인한다.
4. “목적지 말하기” 버튼을 누르고 목적지를 말한다.
5. 앱이 알아들은 목적지를 큰 글자로 보여준다.
6. 사용자가 목적지를 확인한다.
7. 현재 정류장에서 탈 수 있는 버스, 방면, 도착예정시간을 확인한다.
```

목적지 인식 직후 바로 경로를 확정하지 않는다. 같은 이름·유사 발음의 장소가 있으면 사용자가 후보를 선택해야 한다.

## 2. 예외 흐름

| 상황 | 처리 |
|---|---|
| QR에 유효한 관리번호 없음 | 오래된 QR로 안내하고 시민 메인으로 이동 선택 제공 |
| 정류장명은 같지만 방향이 다름 | 관리번호·4자리 번호·방면으로 구별 |
| 마이크 권한 거부·미지원 | 큰 글자 입력과 대표 목적지 선택 제공 |
| 목적지 후보 여러 개 | 후보 이름과 지역·정류장 방면을 표시해 사용자 확인 |
| 운행 노선 없음 | ‘가는 노선을 찾지 못함’ 표시, 임의 대체 금지 |
| TAGO 응답 없음 | 실시간 배지 제거, 검증된 대체정보가 있을 때만 표시 |
| 네트워크 단절 | 저장 정보라는 사실과 기준일 표시, 새 실시간 정보인 것처럼 표현 금지 |

## 3. 행정 운영자

```text
현행 정류장 마스터 확인
  → BIT 설치·운영상태 결합
  → QR 적용 후보 검토
  → 정류장별 QR 생성·부착
  → 현장 스캔으로 정류장명·번호·방면 검수
  → 점검일·교체 이력 기록
```

## 4. 다른 QR과의 구분

- `정류장 진입 QR`: `/qr_main?from={관리번호}`. 현장 출발지를 고정한다.
- `가족 공유 QR`: `?fav={정류장ID 목록}`. 즐겨찾기를 전달한다.

두 QR은 목적, 문구, URL, 테스트를 공유하지 않는다.
