> 작성 2026-07-16. 설계 `docs/specs/2026-07-16-본선-design.md` 기준. 구현 매핑은 계획 `docs/superpowers/plans/2026-07-16-본선.md` 태스크 번호로 표기.
> 표기 원칙(전 흐름 공통): 시설 3상태·출처 배지, "현장 확인"·"수혜" 금지, B2C 점수 숫자 비표시, 미확인은 미확인으로.

# A. B2C — 시민앱 (사용자: 후평동 78세 김OO 님 / 보조 사용자: 자녀·요양보호사)

## A-0. 설계 전제
- 가입·로그인·검색 타이핑 없음. 모든 핵심 흐름은 **탭과 QR 스캔만으로 완결**(키보드로도 조작 가능 — WCAG).
- 화면당 의사결정 1개, 글자 ≥20px, 버튼 ≥56px, 아이콘+한글 병기.
- 데이터는 기기(localStorage)에만. 오프라인은 "최초 1회 접속 후" 동작.

## A-1. 첫 사용 (온보딩 없음) — 기구현
```
접속(URL/QR) → [위치권한 요청]
 ├─ 허용 → 지도 + 현위치 → 가장 가까운 정류장 자동 선택 → 하단 카드
 └─ 거부 → 춘천시청 기준 지도 + 안내 문장 (죽은 화면 없음)
카드 내용: 정류장명 / 도착정보("16번 버스 약 5분" — 실시간 배지 또는 "배차간격 약 N분" 폴백)
          / 시설 4종 3상태+출처 배지("🪑 의자 있음 (벤치대장 기준)" · "❓ 그늘 미확인")
목표: 조작 0회, 3초 내 유용한 정보.
```

## A-2. 정류장 QR로 즐겨찾기 등록 — 신규 (A8)
```
정류장 기둥의 A4 포스터 QR을 폰 카메라로 스캔
 → 브라우저가 앱 열림(?fav=정류장ID — 화이트리스트 검증)
 → 자동으로 즐겨찾기에 병합 (가입·설치·타이핑 없음)
 → 즐겨찾기 화면 + 큰 확인 배너: "○○정류장을 즐겨찾기에 넣었어요"
예외: 이미 등록된 ID → 중복 없이 병합, 배너 동일 / 잘못된 ID → 조용히 무시(화이트리스트)
     오프라인에서 최초 스캔 → 앱 자체가 안 열림(알려진 한계, 발표 시 사전 접속 기기 사용)
```

## A-3. 보호자 대리등록 (원격의 자녀 → 부모 폰) — 기구현
```
[자녀 폰] 앱에서 부모에게 필요한 정류장들 별표 → "가족에게 공유"
 ├─ 네이티브 공유(카톡 등)로 링크 전송, 또는
 └─ QR 표시 → 부모가 카메라로 스캔
[부모 폰] 링크 열림 → ?fav= 목록이 즐겨찾기에 자동 병합 → 대형 즐겨찾기 화면
서버 경유 없음(데이터가 링크 안에 실림). 개인정보 전송 0.
```

## A-4. "버스 타러 가기" — 현재 위치에서 바로 (기구현 + A2 문구)
```
홈 [🚌 버스 타러 가기] → 최근접 정류장 카드
 → 도착정보 확인 → (배차가 길고 4조건 만족 시) 대안 정류장 힌트
 → [여기로 가는 길 안내] → 지도에 도보 경로선 + "도보 약 N분"(OSRM) / 오프라인 "직선거리 약 N분"
```

## A-5. "앉아서 기다리는 길" — 목적지까지 (핵심 신규, A7)
```
홈 [⭐ 자주 가는 곳] 또는 /go → 즐겨찾기 목록에서 목적지 탭 (타이핑 없음)
 → 경로 후보 카드 목록. 상단 토글: [시설 확인된 곳 우선](기본) / [가까운 순]
    부제: "확인된 시설이 있는 길을 우선 보여드려요"
 → 카드(문장 중심, 점수 숫자 없음):
    "① 집에서 4분 걸어서 → ② ㅇㅇ정류장 → ③ 16번 버스 (약 5분) → ④ 병원 앞 하차"
    + 시설별 문구(정확히 이 4종만):
      · 의자 확인      → "앉아서 기다릴 수 있어요 (벤치대장 기준)"
      · 그늘만 확인    → "그늘이 확인된 정류장이에요"
      · 조명만 확인(야간) → "조명이 확인된 정류장이에요"
      · 시설 미확인    → "시설 정보가 아직 확인되지 않았어요"
    + 비교 한 줄: "가까운 곳보다 1분 더 걷습니다"
 → [크게 보기] / [자녀에게 보내기(QR)]
예외: 즐겨찾기 없음 → "먼저 자주 가는 곳을 별표로 저장하세요"
     직행·환승 미도달 → "직접 가는 버스를 찾지 못했습니다" (정직 안내)
     의자 미확인 후보 → "앉아서" 문구 절대 미사용(테스트 강제)
```

## A-6. "지금 기다리기 좋은 곳" — 폭염 시간대 (A2)
```
13~16시 접속 시 홈에 안내: "지금은 그늘 있는 정류장이 편해요" + [그늘 확인 정류장 보기]
 → 기존 시설 필터(그늘=있음)가 켜진 지도 — 확인된 곳만 강조, 미확인은 흐림(강조 제외지 배제 아님)
```

## A-7. 오프라인 (하드 제약 — A12 매트릭스로 검증)
```
[전제: 최초 1회 접속으로 캐시 완료]
오프라인 재접속 → 앱셸+정류장 데이터(캐시)+지도 타일(캐시) 동작
 · 도착정보 → "배차간격 약 N분" 즉시 폴백 (무한 스피너 없음)
 · 도보 경로 → "직선거리 약 N분" + 회색 점선
 · 경로 탐색 → 로컬 routes.json으로 정상 동작
 · 안내 문장: "지금은 저장된 정보로 보여드려요"
```

## A-8. 인쇄물 (행정 접점 — A8·A13)
```
정류장 카드 → [안내문 인쇄] → A4 포스터(초대형 글씨 + 시설 3상태 + QR)
 → 행정이 정류장에 부착 → A-2 흐름의 입구가 됨 (B2C↔B2G 연결 고리)
```

# B. B2G — 행정 대시보드 (사용자: 춘천시 교통행정과·도시재생과 주무관)

## B-0. 설계 전제
- 핵심 질문: "예산을 어디에, 왜, 어떤 순서로?" — 답은 **2단계 정책 흐름**(조사 → 설치 검토).
- 모든 숫자에 출처·기준일 배지("2025.6 4일 표본, 양방향 합산"). CSV는 구성요소 전 컬럼 포함(엑셀 재검산 가능).
- no와 unknown은 절대 같은 트랙에 섞이지 않음(🛑 중단 조건 1).

## B-1. 첫 화면 (A9)
```
/admin 접속 → KPI 4개: 정류장 1,890 / 의자 확인 825 / 그늘 확인 4 / 미확인 조사대상 N
 → 탭: [1단계 조사 검토 순서] [2단계 설치 검토] [조건 필터(v1 보존)]
```

## B-2. 1단계 — 조사 검토 순서 (A3·A9)
```
정책 프리셋 선택: [폭염 대응형] [고령자 이동지원형] [이용량 중심형]
 (각 버튼에 가중치 값 + 근거 한 줄. 인접도(POI)는 고령자형에서만 활성. 슬라이더는 "직접 조정" 보조 패널)
 → 순위표: 순위 | 정류장명 | 한낮 승차(실측) | 미확인 시설 수 | 노선 수(참고) | 선정 사유
    (지수 숫자는 행 끝 보조 셀. "점수"라 부르지 않음)
 → 표 하단 별도 섹션: "수요·시설 모두 미확인 — 조사 후보(순위 없음)" (농촌 223곳 등 — 증발 금지)
 → [조사 목록 CSV] — 현장조사 과업 전달용(로드뷰 URL 포함)
 → 정책 시나리오 비교 표: 프리셋 3종에서 Top10 진입 빈도·평균 순위 ("민감도 분석" 지칭 금지)
```

## B-3. 근거 카드 (행 클릭 — A9)
```
행 클릭 → 근거 카드:
 · 산식 전체 공개 + 항별 분해(실측값 → 분위수 × 가중치)
 · 시간대별 승차 그래프("양방향 합산" 병기) / 시설 4종 3상태+출처 배지 / 인접 시설 거리
 · [CSV 내보내기] [A4 보고서 인쇄] — 의회 답변·보고서 첨부용
```

## B-4. 조사 사이클 — 이 서비스의 척추 (A5 → 파이프라인)
```
① 1단계에서 조사 목록 CSV 내보내기
② [본선 전 실제 수행] 팀원이 로드뷰 캡처(판단 없음) → AI가 판독 초안(yes/no/unclear + 근거)
   → 사람이 검수·확정 (no는 조사 범위가 충분히 보일 때만, 조명은 기본 미확인 유지)
③ 확정 CSV를 행정 조사 입력 형식으로 투입 → build_stops.py 재실행
④ 전후 수치 4종이 대시보드에서 변화:
   투입 전: no=0 · 설치 검토 후보 0곳
   조사 확정본: N곳 조사 — yes A / no B / unknown C
   투입 후: 시설별 설치 검토 후보 수
   제외된 unknown 수 (미확인은 후보가 되지 않았음을 명시)
발표 문장: "본선 전에 사람이 검수한 로드뷰 조사 결과를 행정 조사 입력 형식으로 투입하면,
          설치 검토 목록이 자동 생성됩니다." (※ "현장조사 라이브"처럼 연출 금지)
```

## B-5. 2단계 — 시설별 설치 검토 (A4·A9)
```
[2단계 설치 검토] 탭 → 시설별 서브탭: [벤치] [그늘막] [조명]
 · 대상 = 조사에서 '없음(no, 로드뷰 확인)'이 확정된 정류장만 (yes 기설치·unknown 미확인은 제외)
 · 순위 = 수요 + 인접도 (시설별 별도 표)
 · 전 행 고정 라벨: "데이터상 설치 검토 후보 · 현장 설치 적합성 미검토"
   (보도폭·사유지·도로점용은 데이터 없음 — 최종 판단은 행정 현장 검토)
 · 조사 반영 전(no=0)이면: "조사 반영 전입니다 — 1단계 조사 검토 순서를 먼저 활용하세요"
```

## B-6. 시설별 예산 시뮬레이터 (A9)
```
시설 트랙 선택 → 단가 입력(기본값에 "가정, 편집 가능" 라벨 + 기준일) → 예산 슬라이더
 → 결과: "대상 정류장 N곳 / 표본기간(2025.6, 4일) 한낮 승차 합계 M건
         (양방향 합산, 수요 미확인 K곳 별도)"   ← "수혜"·"일평균" 표현 금지
```

## B-7. 데이터 검증 열람 (A1)
```
대시보드 하단 링크 → docs/데이터_검증.md 기반 검증 요약:
 하루 제외 Top20 안정성 / 동명 조인 충돌 / 결측 223곳 분포 / 원본 20곳 대조 /
 수요-노선 상관(노선 제외 근거) / 기준일 불일치표
용도: "이 순위를 믿어도 되는가"에 대한 선제 답변 (분석 타당성 20점).
```

## B-8. (로드맵 — P2, 화면 미구현) 시민 신호
발표에서 10~15초 한 문장만: *"향후에는 식별정보 없는 시민 반응 집계를 보조 신호로 추가할 수 있지만, 현재 우선순위에는 검증되지 않은 시민 신호를 섞지 않았습니다."*

# C. 흐름 간 연결 (발표 서사용 한 장)

```
[B2C] 김OO 님이 QR로 등록하고 "앉아서 기다리는 길"로 이동
        ↓ 시설 미확인 정류장이 많다는 사실이 화면에 정직하게 노출
[B2G] 행정은 1단계 "조사 검토 순서"로 어디부터 확인할지 결정
        ↓ 조사(AI 보조·사람 확정) → 데이터에 yes/no 확정 반영
[B2G] 2단계 시설별 설치 검토 + 예산 → 설치
        ↓ 설치되면 stops.json 갱신
[B2C] 김OO 님 화면의 "미확인"이 "있음"으로 바뀜 — 선순환 완성
```
