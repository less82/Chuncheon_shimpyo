# 쉼표 정류장 — 사용자 흐름

> 기준: `docs/제품_기획.md`. 상세 화면은 `docs/본선_스토리보드.md`를 따른다.

## 1. 시민 핵심 흐름

```text
정류장 QR 촬영
→ 방향별 현재 정류장 확인
→ 목적지 음성 입력·후보 확인
→ 탈 버스와 예상 대기시간 확인
→ 같은 승차 정류장의 시설상태 확인
→ 선택: 즐겨찾기 / 정보가 달라요
```

즐겨찾기는 기기에만 저장한다. 연령대·이용신호 제공은 선택이며 거부해도 서비스가 동일하게 동작한다.

## 2. 시설 안내

| 상태 | 시민 문구 |
|---|---|
| 있음 | `의자 있음 · 출처와 기준일` |
| 없음 확인 | `의자 없음 확인 · 현장조사일` |
| 미확인 | `의자 미확인 · 확인 가능한 자료 없음` |
| 충돌 | `정보 재확인 중 · 자료와 제보가 다름` |

인근·맞은편 시설은 현재 정류장 시설로 합치지 않으며 이동을 권하지 않는다.

## 3. 제보 흐름

```text
[정보가 달라요]
→ 시설 선택
→ 있어요 / 없어졌어요 / 고장났어요 / 잘 모르겠어요
→ 전송 전 수집항목·목적 안내
→ 검수 대기
→ 행정 화면의 재확인 목록
```

## 4. 행정 흐름

```text
대시보드 개요
→ 1단계 조사 검토 순서
→ 근거 카드(공식수요·시민신호·시설결측·출처)
→ 조사 결과 입력·검수
→ 2단계 시설별 개선 검토
→ CSV·A4 과업표 출력
→ 현장 공사 적합성은 별도 판단
```

## 5. 실패 상태

- QR ID 오류: 가까운 정류장으로 임의 대체하지 않음
- 음성 실패: 큰 글자 입력·대표 목적지 선택
- 도착정보 실패: 실시간 배지 제거, 출처가 있는 대체값만 표시
- 시설자료 없음: `미확인`
- 시민 이벤트 전송 실패: 안내 기능은 계속 제공하고 개인정보를 재전송 큐에 장기 보관하지 않음
