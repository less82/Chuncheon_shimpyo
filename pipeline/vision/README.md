# 시설 탐지 라벨링 · 학습 (로컬 전용, Roboflow 불필요)

`docs/specs/2026-07-22-시설변화감지-AI-design.md` §7 구현.
**손으로 박스를 그리지 않는다.** 오픈보캡 모델이 자동 생성 → 사람은 **틀린 것만 수정**.

전 과정 로컬. 외부 API 호출·과금·토큰 0.

## 클래스 (id 고정 — 바꾸면 기존 라벨 전부 무효)

| id | name | 한글 | 포함 | 제외 |
|---|---|---|---|---|
| 0 | `roof` | 지붕/차양 | 정류장 상부 구조물, 쉘터 지붕, 차양 | **나무 그늘** |
| 1 | `bench` | 벤치 | 대기용 벤치·의자 | 상점 밖 의자 등 정류장 무관 |
| 2 | `bit` | BIT 전광판 | 버스도착정보 LED/LCD 단말 | 노선도 종이판, 광고판 |

규칙: **인스턴스마다 박스 1개. 50% 이상 가려지면 스킵.**

## 실행

```bash
cd pipeline
V=.venv-vision/Scripts/python.exe

# 1) 오토라벨 (162장, GPU 몇 분)
$V autolabel.py --preview

# 2) 검수 — 브라우저 편집기 (표준 라이브러리만, venv 아니어도 됨)
python vision_review.py
#    1/2/3 클래스 · 드래그 새 박스 · 클릭 선택 · Del 삭제
#    ←/→ 이동(자동저장) · S 저장 · V 저장+검수완료

# 3) 학습 → 가중치 export
$V train_facility.py
```

## 산출물

| 경로 | 내용 |
|---|---|
| `autolabel_raw/*.txt` | 자동생성 원본. **불변** — 검수 전후 대조·감사용 |
| `labels/*.txt` | 검수 대상 라벨(YOLO 포맷). 편집기가 여기를 씀 |
| `preview/*.png` | 자동라벨 눈검사용 오버레이 |
| `splits.json` | train/val 분할(시드 고정 20260722). val 50장 |
| `review_state.json` | 사진별 검수완료 여부 |
| `dataset/` | 학습 직전 재구성(자동, 지워도 됨) |
| `weights/facility_v1.pt` | 로컬 추론용 가중치 → `facility_vision.py`가 로드 |
| `metrics.json` | 검증셋 클래스별 mAP |

## 순환논리 금지 (없앨 수 없는 최소 수작업)

`splits.json`의 **val 50장은 자동라벨을 믿으면 안 된다.** 자동라벨로 잰 정확도는
자기 자신을 채점하는 것이라 발표에 쓸 수 없다. 편집기에서 `검증` 뱃지가 붙은 사진은
**전수 손검수**해야 하고, `train_facility.py`가 미검수 상태면 학습을 거부한다.

train 112장은 전수 검수가 아니어도 학습이 돌아간다(라벨 잡음은 증강으로 어느 정도 흡수).
다만 BIT 클래스는 오탐/누락이 잦으니 우선 훑을 것.
