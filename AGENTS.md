# 쉼표 정류장 — 팀 협업 규칙 (AI 에이전트 공용)

> 이 문서는 **Codex(`AGENTS.md`)와 Claude Code(`CLAUDE.md`)가 함께 읽는 단일 규칙서**입니다.
> Claude Code용 `CLAUDE.md` 는 이 파일을 참조합니다. 규칙을 바꿀 땐 **이 파일만** 고치세요.

## 1. 브랜치 규칙 — 사람별 브랜치

- 각자 **자기 이름 브랜치**에서만 작업합니다: `kook`, `oh`, `princess-work` 등.
- **남의 브랜치·`main` 에 직접 커밋/푸시 금지.**
- 새 작업도 원칙적으로 자기 브랜치에서 진행합니다(필요하면 `feature/이름-작업` 도 허용, 단 자기 것만).

## 2. 커밋·푸시 규칙 — "작업 끝날 때마다 자기 브랜치에 푸시"

- **작업이 일단락되면 자기 브랜치에 커밋하고 즉시 `git push`** 합니다. (백업 + 팀 가시성)
- 커밋 메시지는 한국어로, 무엇을 왜 바꿨는지 한 줄 요약.
  - 예: `feat(map): 도보 경로선 추가`, `fix(trip): 오프라인 경로탐색 폴백 수정`
  - 자동 저장 커밋은 `wip(브랜치): 자동저장 …` 형식(아래 4절) — PR에서 squash되므로 신경 안 써도 됨.
- **절대 하지 말 것:** `main`/`master` 로의 직접 push, `git push --force` (자기 브랜치 외), 남의 브랜치 push.

## 3. main 병합 규칙 — PR 리뷰로만

1. 자기 브랜치 작업 완료 → 자기 브랜치 push
2. GitHub에서 **자기 브랜치 → `main` Pull Request** 생성
3. 팀원 1명 이상 리뷰/확인 후 병합 (squash merge 권장 — main 이력이 깔끔해짐)
4. `main` 에 직접 머지·푸시하지 않습니다.

## 4. 자동 저장 훅 (Claude Code 사용자)

Claude Code는 응답을 마칠 때마다 `scripts/auto-push.sh` 를 실행해 **자기 브랜치에 자동 커밋+푸시**합니다.
안전 가드 3종이 있습니다:

1. **`main`/`master` 브랜치면 자동 푸시 안 함** (공유 브랜치 보호)
2. **변경사항 없으면 커밋 안 함** (빈 커밋 방지)
3. **푸시 실패해도 로컬 커밋은 유지** (작업 흐름 안 막음)

- 설정 위치: `.claude/settings.json` 의 `Stop` 훅 (팀 공용, 커밋됨).
- **끄고 싶으면:** `.claude/settings.local.json`(개인·gitignore) 에 `{ "disableAllHooks": true }` 또는 `/hooks` 메뉴에서 비활성화.

### Codex 사용자 (동일 동작 맞추기)
Codex는 위 Stop 훅을 읽지 않으므로, 같은 "작업 끝날 때 자기 브랜치 푸시"를 맞추려면 둘 중 하나:
- **수동:** 작업 끝낼 때 `bash scripts/auto-push.sh` 를 직접 실행(스크립트는 에이전트 무관하게 동작).
- **자동:** Codex 쪽 훅/후처리 설정에 `bash scripts/auto-push.sh` 를 등록.
- 스크립트는 현재 브랜치를 자동 판별하므로 kook/oh 누구든 안전하게 씁니다.

## 5. 충돌 예방

- 같은 파일을 둘이 동시에 만지면 병합 시 충돌 → 작업 시작 전 채팅으로 "나 지금 map 쪽 만진다" 공유.
- 자기 브랜치는 수시로 `git fetch origin main && git merge origin/main` 으로 main 최신분을 받아 충돌을 작게 유지.

## 6. 절대 규칙 (보안·정직성 — 프로젝트 정체성)

- **비밀키 커밋 금지:** TAGO 키는 `app/.env`(gitignore됨)에만. 채팅·커밋·PR에 키 붙여넣지 않음.
- 시설 데이터는 **3상태(있음/없음/미확인)**, 근거 없는 "없음" 생성 금지 — 자세한 건 `docs/데이터_현황.md`.
- `stops.json` 스키마(계약)는 `app/src/types/stop.ts` — 바꾸려면 팀 합의.

_요약: **자기 브랜치에 자유롭게(자동) 푸시 → main은 PR 리뷰로만.** 이게 전부입니다._
