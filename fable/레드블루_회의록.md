## 2. 2라운드 쟁점별 수렴 결과

### 쟁점 A — 폭염 민원 0건인데 폭염 시나리오가 맞는가?
**합의: 폭염 v1 유지, 단 서사 층위 변경.** 폭염을 '문제의 근거'에서 퇴출하고 '시연 레이어 선택 이유'로 강등. 근거는 "시설 민원 연중 12건 + 6월 온열의자". 원칙은 **선제 공개** — 0건은 심사위원이 발견하면 탈락 사유, 우리가 먼저 공개하면 정직성의 증거. 방어 논리: "민원은 시가 고칠 수 있다고 믿는 것만 기록된다. 시민은 '덥다'가 아니라 '의자를 고쳐 달라'고 넣는다. 폭염은 문제가 '불편'에서 '건강·안전'으로 격상되는 증폭기다."

### 쟁점 B — 쉘터 데이터 공백 (레드팀: 탈락 사유급)
**합의: 3중 대응으로 '정직한 한계 관리의 모범 사례'로 전환.**
1. **3상태 표기**: [있음(확인)/없음(확인)/미확인]. 근접 매칭만으로 '없음' 단정 금지 → 신뢰 붕괴 경로 구조적 제거.
2. **로드뷰 실사 150곳 사전 구축** (경유 노선 수 상위 기준, 3인 분담 ~1시간) → '현장 확인' 배지, 시연 동선은 검수군에서만. 자체 구축 데이터 2호로 데이터 활용성 가산.
3. **시민 제보 버튼** — 공백을 "시민 참여 성장형 데이터셋" 서사로 전환, 6절 공개 요청과 연결해 "자백"을 "데이터 개방 효용의 실증 제안"으로 재프레임.

### 쟁점 C — 대안 정류장 기능 (버스 놓침 책임)
**합의: 4중 안전장치 + 3단 사다리.** ① 동일 노선 ② 직선 300m 이내(분속 60m 보수 환산, 직선거리 명시) ③ 시설 우위(미확인 항목 판정 배제) ④ 도보시간 ≤ 배차간격 1/2(확인 불가 시 비활성). 배차 확보: TAGO 정적 캐시 → 공통 경유 노선 수 대리 지표("공통 노선 ≥ 2 AND 도보 ≤ 4분") → 최후엔 "시설 비교 정보"로 라벨 강등. 포지셔닝은 **출발 전 대기 계획 도구**.

### 기타 합의
- 부제: "사계절 버스정류장 대기환경 진단·안내 플랫폼 (v1 시연: 폭염 시나리오)". 서비스명 '쉼표 정류장' 유지.

## 4.5. 추가 검토 (2026-07-03) — "12건이면 주제를 노선 불편 처리로 바꿔야 하지 않나"

제출일에 팀 내부에서 제기된 주제 교체 제안("버스 노선 불편사항 처리")을 데이터로 검증 후 기각했다.

- **실측**: 민원 1,000건 키워드 재분류 결과 노선·배차·운행시간 불편 **23건**(그중 2건은 2026년 파업 관련 일시 이슈) vs 정류장 시설 관련 16~17건(보수적 12건). 민원 건수 체급이 사실상 동일 — 갈아탈 우위가 없음.
- **기각 사유**: ① 노선 주제는 4일 표본 승하차 데이터가 '수요 대표성'이라는 핵심 근거가 되어 치명타(현 주제에선 보조축이라 버팀) ② 배차·운행비용 데이터 비공개 → 산출물이 서비스가 아닌 제안서 → 본선 기술 구현력 40 + 서비스성 20에서 불리 ③ 노선 최적화는 1R에서 이미 기각된 최다 겹침 영역.
- **수용한 부분**: "12건" 인상 리스크는 실재 → 개발기획서 추진배경에 **"왜 노선·배차가 아니라 정류장 대기환경인가"** 선제 방어 문단 추가(노선 23건 비교 포함), 읽기용 예상 질문에 해당 문답 추가.

**외부 데이터 재검증 (2026-07-03, 2차)** — "인터넷에서 구할 수 있는 데이터 + 6절 요청 데이터를 포함하면 노선 시뮬레이터가 가능한가"를 웹 검증한 결과, **기각 유지 + 우리 주제에 유리한 신규 사실 2건 발견**:
1. **OD 데이터는 해커톤에서 사용 불가**: 교통카드 빅데이터(STCIS)의 원시데이터는 데이터안심구역(인터넷 미연결 폐쇄 공간)에서만 분석 가능하고 분석 결과만 반출 허용 — 무박 2일 해커톤장 반입 자체가 불가. 웹 공개 통계의 정류장간 OD는 읍면동 단위 집계라 노선 단위 효과 계산에 부족.
2. **결정타 — 춘천시는 2025.11.28에 노선 조정을 이미 시행**(교통포털 노선도 기준). 노선 변경 효과 시뮬레이터는 시가 방금 마친 일을 다시 하자는 제안이 되고, 우리 승하차 표본(2025.6)은 조정 이전 네트워크의 수요라 시뮬레이션의 기반 자체가 구버전 — 노선 주제는 데이터 시점에서도 성립 불가.
